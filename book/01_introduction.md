# 第1章 はじめに — なぜ仮想環境でラズパイ開発なのか

## 1.1 Raspberry Piとは

Raspberry Pi（ラズベリーパイ、通称「ラズパイ」）は、イギリスのRaspberry Pi財団が開発した、手のひらサイズの小型コンピュータです。価格は数千円程度と安価でありながら、Linux OSが動作し、GPIO（General Purpose Input/Output）と呼ばれるピンを通じてLEDやセンサーなどの電子部品を制御できます。

この特徴により、Raspberry Piは以下のような用途で世界中で広く使われています。

- **IoTデバイス**: 温度や湿度を計測してクラウドに送信するセンサーノード
- **ホームオートメーション**: 照明や空調を自動制御するスマートホームの中核
- **ロボティクス**: モーターやサーボを制御するロボットの頭脳
- **教育**: プログラミングや電子工学の学習ツール
- **プロトタイピング**: 新しいハードウェア製品のプロトタイプ開発

## 1.2 IoT（Internet of Things）とは

IoTとは「モノのインターネット」と訳され、様々な物理デバイスがインターネットに接続され、データを送受信する仕組みを指します。

身近な例を挙げると：

- スマートウォッチが心拍数をスマホに送信する
- エアコンが外出先からスマホで操作できる
- 農業用センサーが土壌の水分量を自動で計測する
- 工場の機械が稼働状況をリアルタイムでクラウドに報告する

IoTシステムは、大きく分けて以下の4つの要素で構成されます。

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ センサー │───>│ デバイス │───>│ 通信    │───>│ アプリ  │
│ (計測)   │    │ (処理)  │    │ (送受信)│    │ (表示)  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
  温度計         Raspberry Pi     MQTT          ダッシュボード
  湿度計                          HTTP          スマホアプリ
  カメラ                          WebSocket     AI分析
```

本書では、これら4つの要素すべてを仮想環境で構築します。

## 1.3 従来の開発方法の課題

Raspberry Piを使ったIoT開発には、通常以下のものが必要です。

### ハードウェア
| 部品 | 価格目安 | 用途 |
|------|---------|------|
| Raspberry Pi 4/5 本体 | 8,000〜15,000円 | メインコンピュータ |
| microSDカード | 1,000〜2,000円 | OSの記録媒体 |
| 電源アダプター | 1,000〜2,000円 | 電源供給 |
| ブレッドボード | 500〜1,000円 | 部品を差し込む基板 |
| LED、抵抗、ジャンパー線 | 500〜1,500円 | 電子部品 |
| 各種センサー | 1,000〜5,000円 | 温度・湿度・距離など |
| サーボモーター | 500〜2,000円 | 角度制御モーター |

合計すると、最低でも1万円以上の初期投資が必要です。さらに以下の課題があります。

**物理的な制約**
- 配線ミスで部品を壊すリスクがある
- 電子部品の到着を待つ必要がある
- 作業スペースが必要

**開発効率の問題**
- Raspberry Pi上での開発は、PCと比べて動作が遅い
- SSHでの遠隔操作はセットアップが面倒
- デバッグが難しい（物理的な状態を確認する必要がある）

**チーム開発の困難さ**
- メンバー全員がRaspberry Piを持つ必要がある
- 同じ配線構成を再現する必要がある
- CI/CDパイプラインに組み込みにくい

## 1.4 仮想環境という解決策

本書で紹介する仮想環境アプローチは、これらの課題をすべて解決します。

### 使用するツール

**PI-CI（ピーアイ・シーアイ）**
- Dockerコンテナ内でQEMUエミュレータを使い、本物のRaspberry Pi OSを起動する
- 実際のRaspberry Pi OS（Bookworm）がそのまま動作する
- Pi 3/4/5に対応
- GitHubで738スター獲得の人気プロジェクト

**gpiozero（ジーピーアイオー・ゼロ）**
- Raspberry Pi公式のPythonライブラリ
- LEDやボタン、センサーなどを簡単に制御できる
- 「MockFactory」機能により、実機がなくてもGPIOをシミュレーション可能
- 本物のRaspberry Piでもまったく同じコードが動作する

**Mosquitto（モスキート）**
- MQTTプロトコルのブローカー（メッセージ中継サーバー）
- IoTの世界で最も広く使われている通信プロトコルのひとつ
- Docker公式イメージが提供されている

### 仮想環境のメリット

| 項目 | 実機 | 仮想環境 |
|------|------|---------|
| 初期費用 | 1万円以上 | 0円 |
| セットアップ時間 | 数時間 | 数分 |
| 部品破損リスク | あり | なし |
| 環境の再現性 | 低い | 完全に再現可能 |
| チーム開発 | 困難 | 容易 |
| テスト自動化 | 困難 | 容易 |
| 実機への移行 | — | 最小限の変更で可能 |

## 1.6 本書で作るもの

本書を通じて、最終的に以下のシステムを構築します。

**15種類のIoTデバイスのシミュレーション**
- LED（デジタル、PWM調光、RGB）
- ブザー
- サーボモーター
- DCモーター
- ボタン
- モーションセンサー（人感センサー）
- ラインセンサー
- 温度・湿度センサー
- 気圧センサー
- 照度センサー
- 超音波距離センサー
- 土壌水分センサー

**3つの通信プロトコル**
- MQTT（リアルタイムデータ配信）
- REST API（Webベースの制御）
- WebSocket（双方向リアルタイム通信）

**実用的なIoTアプリケーション**
- スマート農業システム（自動灌漑）
- スマートドアロック（サーボ制御）
- スマートホーム（照明・空調自動制御）
- データロガー（統計分析・異常検出）
- Webダッシュボード（リアルタイム監視・操作）
