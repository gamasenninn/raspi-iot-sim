# 第8章 複数デバイスの連携制御

この章では、センサーの値に基づいて複数のアクチュエータを自動制御する、実用的なIoTアプリケーションを構築します。

## 8.1 IoT制御の基本パターン

### センス → シンク → アクト

IoTシステムの基本的な制御ループは以下の3ステップです。

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ Sense   │───>│ Think   │───>│ Act     │
│ (計測)  │    │ (判断)  │    │ (実行)  │
└─────────┘    └─────────┘    └─────────┘
  温度=28°C     28>26 → 暑い    ファンON
  湿度=85%      85>80 → 多湿    除湿アラーム
  距離=15cm     15<30 → 近い    照明ON
```

## 8.2 実践: スマートホーム自律制御

### 制御ルール

| 条件 | アクション |
|------|----------|
| 照度 < 200 lux | 室内照明ON（明るさを照度に反比例） |
| 温度 > 26°C | ファン(モーター)ON（速度を温度に比例） |
| 温度 < 18°C | 暖房警告LED ON |
| 湿度 > 85% | 除湿アラームON |
| 距離 < 30cm | 人感検知 → 全照明MAX |

### 実行結果

```
[ 1] T: 21.6°C H: 53.8% L: 519.5lx D:   3.2cm → 人感→全灯 | 空調:OFF
...
[14] T: 20.2°C H: 58.8% L: 546.6lx D:  32.9cm → 照明:OFF | 空調:OFF
```

距離センサーが30cm以下の間は人感検知モードで全灯、離れると自動消灯。複数のセンサー値を組み合わせた判定が正しく動作しています。

## 8.3 実践: スマート農業システム

### システム構成

```
土壌水分センサー ──> 灌漑ポンプ(モーター)制御
距離センサー     ──> 侵入警報(ブザー)
温度/湿度センサー ──> MQTT経由でデータ配信
```

### 灌漑制御のロジック

```python
SOIL_DRY = 40.0   # %以下で灌漑開始
SOIL_WET = 70.0   # %以上で灌漑停止

if soil.value < SOIL_DRY and not irrigating:
    pump.forward(0.8)    # ポンプ稼働（80%速度）
    irrigating = True
elif soil.value > SOIL_WET and irrigating:
    pump.stop()          # ポンプ停止
    irrigating = False
```

この「ヒステリシス制御」は、開始しきい値（40%）と停止しきい値（70%）にギャップを設けることで、境界値でのチャタリング（ON/OFFの高速繰り返し）を防止しています。

## 8.4 実践: スマートドアロック

### シナリオベースの制御

```
1. 人が接近 → モーション検知 → 施錠中のため通知音
2. ボタン解錠 → サーボ max → LED ON
3. 自動施錠タイマー → サーボ min → LED OFF
4. 不審な動き → 警報発動
```

### イベント駆動のリセット

```python
def on_button_press():
    if locked:
        unlock_door()  # サーボ → max
    else:
        lock_door()    # サーボ → min

btn.when_pressed = on_button_press
```

gpiozeroのコールバック機能（`when_pressed`）を使うことで、ボタンが押された瞬間に即座に反応する「イベント駆動」の制御を実現しています。

## 8.5 設計パターンまとめ

本章で学んだIoT制御パターン：

1. **しきい値制御**: センサー値が一定値を超えたらアクション実行
2. **ヒステリシス制御**: 開始/停止の2つのしきい値でチャタリング防止
3. **優先度制御**: 人感検知が他の制御ルールより優先
4. **イベント駆動**: コールバックによるリアルタイム応答
5. **シナリオ制御**: 一連の操作を順序立てて実行
